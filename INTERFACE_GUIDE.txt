KEY FEATURES
================================================================================

1. DIAGNOSTICS PANEL
   ─────────────────
   Location:    Top-right corner, left of "Connected" status
   Trigger:     Click "Diagnostics" button
   State:       Hidden by default
   Close:       Click button again or click outside panel
   
   Displays:
   • FPS (frames per second)
   • Inference time (milliseconds)
   • Total frames processed
   • Active connections

2. FRESHNESS TRACKING
   ──────────────────
   Tracked Products:
   • Passion fruit
   • Maui custard
   • Lemon cake
   • Mango
   • Watermelon
   • Pineapple
   
   Display Format:
   Fresh:    "Fresh - X days old" (gray text)
   Expired:  "EXPIRED (X days old)" (red text)
   
   Expiration:
   • Products expire after 5 days
   • Timer starts when first detected
   • Updates automatically
   • Persists across page views

3. SALES LOG
   ─────────
   Detection:   Every 5 seconds
   Method:      Compares inventory totals
   Logic:       Total decrease = sale(s)
   
   Entry Format:
   Product Name          MM/DD/YYYY, HH:MM:SS AM/PM
   
   Features:
   • Chronological order (newest first)
   • US Eastern Time (EST)
   • Scrollable list
   • Sales counter badge
   • Persistent during session

================================================================================
COLOR CODING
================================================================================

Status Indicators:
  ● Green dot     = Connected
  ● Gray dot      = Disconnected

Inventory:
  Green numbers   = Item counts
  Green border    = Active inventory item
  Gray text       = Fresh products (< 5 days)
  Red text        = Expired products (≥ 5 days)

Sales Log:
  Blue border     = Sales entry
  Blue badge      = Sales counter

Diagnostics:
  Green text      = Metric values
  Green border    = Diagnostics button

================================================================================
USER INTERACTIONS
================================================================================

Diagnostics Panel:
  1. Click "Diagnostics" button → Panel appears
  2. Click "Diagnostics" again → Panel closes
  3. Click anywhere outside → Panel closes

Inventory List:
  • Hover over item → Slight highlight
  • Scroll to view all products
  • Sorted by count (highest first)

Sales Log:
  • Hover over entry → Background darkens
  • Scroll to view history
  • Newest entries at top

================================================================================
TECHNICAL DETAILS
================================================================================

Freshness Calculation:
  • Timestamp stored when product first detected
  • Age = Current time - First seen time
  • Displayed in days (rounded down)
  • Recalculated on every inventory update

Sales Detection:
  • Check interval: 5 seconds
  • Previous total: 30 items
  • Current total: 28 items
  • Result: 2 sales recorded

Time Zone Conversion:
  • All times displayed in US Eastern Time (EST)
  • Format: MM/DD/YYYY, HH:MM:SS AM/PM
  • Example: 01/10/2026, 10:15:32 AM

CUSTOMIZATION

Expiration Threshold:
  • Change EXPIRATION_DAYS constant (line ~543)
  • Default: 5 days

Sales Check Interval:
  • Change SALES_CHECK_INTERVAL constant (line ~546)
  • Default: 5000 ms (5 seconds)

Tracked Products:
  • Edit TRACKED_PRODUCTS array (line ~542)
  • Add/remove product names (lowercase)

================================================================================
TROUBLESHOOTING
================================================================================

Issue: Diagnostics panel not showing
Fix:  • Check browser console for errors
      • Verify WebSocket connected
      • Hard refresh page

Issue: Freshness timers missing
Fix:  • Verify product name matches tracked list
      • Check case-insensitive matching
      • Ensure product detected in inventory

Issue: Sales not recording
Fix:  • Verify inventory total decreasing
      • Wait 5 seconds between checks
      • Check browser console logs

Issue: Wrong time zone
Fix:  • Verify browser supports Intl.DateTimeFormat
      • Check system time zone settings
      • Confirm 'America/New_York' available

Issue: Interface not updating
Fix:  • Check WebSocket connection status
      • Verify backend server running
      • Check network connectivity
